Why Apple Silicon? Steve AI runs a 7-billion-parameter code model entirely on your Mac's Neural Engine and GPU. This requires the unified memory architecture available only on M-series chips.

Installation

Download Kesef Kode from the TestFlight beta or the Mac App Store. On first launch, Steve's code model (~4 GB) is loaded from the bundled model directory. No API keys, no account required for core features, no internet connection needed after the initial download.

Welcome Screen

When no project is open, you'll see the Welcome Screen with four actions:

  • Open Folder — browse to any directory on your Mac
  • Clone Repository — enter a Git HTTPS or SSH URL and choose a destination
  • New Project — scaffold a SwiftUI project for any Apple platform (iOS, macOS, watchOS, tvOS, visionOS)
  • Recent Projects — quick access to your 10 most recently opened projects

The IDE

A full-featured native macOS IDE built with SwiftUI and AppKit, designed around the "Liquid Glass" aesthetic — translucent backgrounds, vibrant gradients, and fluid animations.

File Navigator

The left sidebar displays your project's file tree with live sync — the tree automatically refreshes when files change on disk. Smart filtering hides DerivedData, .build, .git, node_modules, and Pods. Git status badges appear next to modified, added, and deleted files.

Code Editor

The central editor features syntax highlighting for 15+ languages (Swift, Objective-C, Python, JavaScript, TypeScript, HTML, CSS, JSON, YAML, Markdown, and more), configurable line numbers, word wrap, and tab management with unsaved change indicators.

  • LSP Integration — powered by SourceKit-LSP for real-time diagnostics, code completion, and document sync
  • Find & Replace — ⌘F for find, ⌘H for find-and-replace
  • Auto-save — dirty files saved 2 seconds after last edit (configurable)
  • Unsaved changes protection — closing a dirty tab prompts Save / Don't Save / Cancel

Integrated Terminal

Toggle with ⌘`. A fully interactive zsh session rooted at your project directory with live streaming output, input support for interactive processes, and a kill button for running processes.

Build & Run

Press ⌘R to build and run. Kesef Kode saves all dirty files, auto-detects your project type (.xcworkspace, .xcodeproj, or Package.swift), detects the scheme, builds with xcodebuild, and launches the built .app.

Source Control

Built-in Git integration with branch display, changed file count, commit (⌘⌥C), push, pull, and per-file status tracking. All Git operations are routed through the secure ProcessRunner, preventing shell injection attacks.

Steve AI

Steve is the on-device AI coding engine running a custom-fine-tuned Qwen 2.5 Coder 7B model via Apple's MLX framework — entirely on your Mac's hardware.

What Steve Can Do

  • Read your project structure and understand file relationships
  • Write complete, compilable Swift files with // File: headers
  • Execute shell commands (builds, git, tests) and reason about the output
  • Search the web when he needs up-to-date information
  • Remember facts about you and your project across conversations
  • Retrieve relevant Apple documentation from a 1.6M-page local database

Fast Mode

The default mode. Your message is enriched with project context, RAG documentation, memories, and skills. Steve streams a response with code blocks that can be auto-applied to your project with one click.

Plan Mode

Toggle to Plan mode for complex, multi-step tasks. Steve analyzes your request, generates a structured plan (max 8 steps), presents it for your approval, then executes code-only output after you approve.

No repetition loops. Steve's plan is surgically removed from conversation history before execution, ensuring he writes code instead of re-explaining.

Inline Editing — ⌘K

Press ⌘K in the code editor to invoke Steve for inline modifications. Type an instruction like "add error handling" or "convert to async/await" — Steve receives the full file content plus your instruction and returns the modified file in-place.

Vision AI

Drag and drop a UI mockup screenshot into the Steve chat panel. A local Phi-3 Vision 4-bit model analyzes the layout, colors, typography, and components — then generates matching SwiftUI code. The Vision model (~2.2 GB) is loaded on-demand and swapped with the code model to conserve memory.

Web Research

When Steve encounters a query requiring up-to-date information, he automatically searches DuckDuckGo, fetches top pages via a headless WKWebView, extracts text content, and injects results into his prompt. Fully sandboxed and Mac App Store compatible.

RAG System

Retrieval-Augmented Generation is what makes a 7B model punch above its weight. Kesef Kode ships with a pre-built SQLite database containing 1.6 million+ scraped Apple Developer documentation pages.

How It Works

  1. When you ask Steve a question, the RAG service searches the documentation using FTS5 full-text search with BM25 ranking
  2. Query expansion adds Apple developer synonyms (e.g., "navigation" → NavigationStack, NavigationLink, NavigationPath)
  3. The top 3–5 most relevant document chunks are injected into Steve's prompt
  4. Steve's response is grounded in real SDK documentation, not hallucinated API surfaces

Supported Frameworks

SwiftUI, UIKit, AppKit, Combine, Core Data, SwiftData, RealityKit, WidgetKit, async/await concurrency, networking (URLSession), storage (FileManager, Keychain), and more — all with automatic query expansion.

Zero extra RAM. The database uses SQLite memory-mapping. Searches take <10ms on SSD.

Memory System

Steve remembers facts about you across conversations. When he learns something important — your coding style, project architecture, preferences — he emits a [REMEMBER: ...] marker that's automatically parsed, categorized, and saved to your Kesef Plus account via Supabase.

Advanced Features

MCP Servers

Kesef Kode supports the Model Context Protocol (MCP) — an open standard for connecting AI systems with external tools. Add servers in Settings → MCP or configure via ~/.kesef-kode/mcp_config.json. Steve can invoke MCP tools during conversation, and all requests have a 30-second timeout to prevent freezing.

Custom Skills

Skills are markdown files that inject domain knowledge into Steve's prompt when triggered by keywords. Place .md files in ~/.kesef-kode/skills/ (global) or {project}/.kesef/skills/ (workspace-specific) with optional YAML front-matter for triggers and descriptions.

Simulator Manager

Queries xcrun simctl for all available Apple simulators. View, boot, install, and launch apps on any iOS, watchOS, tvOS, or visionOS simulator device.

LLDB Debugger

An integrated LLDB session for low-level debugging. Send commands directly (breakpoint set, po, thread backtrace) with live-streamed output.

Conversation Management

Multiple conversation threads with save, load, and delete. Conversations are persisted to ~/.kesef-kode/conversations/ as JSON with full message history and original timestamps.

Settings

Access via Kesef Kode → Settings (⌘,). Six tabs:

  • General — auto-save interval, default AI mode (Fast/Plan), startup behavior
  • Editor — font size (10–24pt), tab size, line numbers, word wrap, minimap
  • Steve AI — model info, auto-suggest toggle, max tokens (1024/2048/4096), temperature slider (0.1–1.0)
  • MCP — add/remove MCP servers, view connected status and tools
  • Skills — view, create, and delete custom skills
  • Advanced — thermal management, system RAM, LLDB/LSP paths, reset all settings

Architecture

Kesef Kode is built as a native macOS application with a service-oriented architecture.

Key Design Decisions

  • On-device only — all inference runs via Apple's MLX framework. Zero data leaves your Mac.
  • Model swapping — only one model in memory at a time. Steve and Vision are swapped on demand with GPU cache cleared between swaps.
  • Context window management — the prompt builder enforces a 12,000-character budget (~4K tokens) with priority-based truncation.
  • Safety — all AI-initiated shell commands pass through ProcessRunner's blocklist. Code application has path traversal protection.

Privacy & Security

On-Device Processing

All AI inference runs on your Mac's Apple Silicon. Your code, prompts, and project files never leave your device. No telemetry, no cloud processing, no third-party AI API calls.

Shell Execution Safety

Every AI-initiated command passes through ProcessRunner's safety layer, blocking privilege escalation (sudo), destructive filesystem ops (rm -rf /), remote code execution (curl | sh), fork bombs, and disk wipe commands.

Path Traversal Protection

When Steve writes code to your project, filenames are sanitized — .. sequences stripped, absolute paths rejected, and resolved paths verified to be within the project directory.

FAQ

Does Kesef Kode require internet?

No. All AI inference runs locally. Internet is only used for: initial model download, web research (when Steve needs up-to-date info), optional memory sync via Kesef Plus account, and Git push/pull.

What Apple platforms can I build for?

All of them: iOS, iPadOS, macOS, watchOS, tvOS, and visionOS. Kesef Kode uses xcodebuild under the hood and supports workspace, project, and Swift Package Manager project types.

Can I use my own models?

Custom model support is planned for a future release.

What is the context window size?

Steve operates within an ~8K token context window. The prompt builder automatically manages a 12,000-character budget, prioritizing your active file and message while truncating lower-priority context to fit.

How do I report a bug?

Use Help → Report a Bug in the menu bar, or Help → Send Feedback to open a GitHub issue with a pre-filled template.
