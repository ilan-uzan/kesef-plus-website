Why Apple Silicon? Steve AI runs a proprietary code model entirely on your Mac's Neural Engine and GPU. This requires the unified memory architecture available only on M-series chips.

Installation

Download the latest DMG from GitHub Releases or the download page. On first launch, Steve's code model (~4 GB) is loaded from the bundled model directory. No API keys, no account required for core features, no internet connection needed after the initial download.

Welcome Screen

When no project is open, you'll see the Welcome Screen with four actions:

  • Open Folder — browse to any directory on your Mac
  • Clone Repository — enter a Git HTTPS or SSH URL and choose a destination
  • New Project — scaffold a SwiftUI project for any Apple platform (iOS, macOS, watchOS, tvOS, visionOS)
  • Recent Projects — quick access to your 10 most recently opened projects

The IDE

A full-featured native macOS IDE built with SwiftUI and AppKit, designed around the "Liquid Glass" aesthetic — translucent backgrounds, vibrant gradients, and fluid animations.

File Navigator

The left sidebar displays your project's file tree with live sync — the tree automatically refreshes when files change on disk. Smart filtering hides DerivedData, .build, .git, node_modules, and Pods. Git status badges appear next to modified, added, and deleted files.

Code Editor

The central editor features syntax highlighting for 15+ languages (Swift, Objective-C, Python, JavaScript, TypeScript, HTML, CSS, JSON, YAML, Markdown, and more), configurable line numbers, word wrap, and tab management with unsaved change indicators.

  • LSP Integration — powered by SourceKit-LSP for real-time diagnostics, code completion, and document sync
  • Find & Replace — ⌘F for find, ⌘H for find-and-replace
  • Auto-save — dirty files saved 2 seconds after last edit (configurable)
  • Unsaved changes protection — closing a dirty tab prompts Save / Don't Save / Cancel

Integrated Terminal

Toggle with ⌘`. A fully interactive zsh session rooted at your project directory with live streaming output, input support for interactive processes, and a kill button for running processes.

Build & Run

Press ⌘R to build and run. Kesef Kode saves all dirty files, auto-detects your project type (.xcworkspace, .xcodeproj, or Package.swift), detects the scheme, builds with xcodebuild, and launches the built .app.

Source Control

Built-in Git integration with branch display, changed file count, commit (⌘⌥C), push, pull, and per-file status tracking. All Git operations are sandboxed through a secure execution layer, preventing shell injection attacks.

Steve AI

Steve is the on-device AI coding engine running a proprietary, Apple-optimized code model — entirely on your Mac's hardware. No cloud, no API keys, no subscriptions.

What Steve Can Do

  • Read your entire project structure and understand file relationships
  • Write complete, compilable Swift files and apply them to your project
  • Execute shell commands (builds, git, tests) and reason about the output
  • Search the web when he needs up-to-date information
  • Remember facts about you and your project across conversations
  • Retrieve relevant Apple documentation from a massive local knowledge base

Fast Mode

The default mode. Your message is enriched with project context, RAG documentation, memories, and skills. Steve streams a response with code blocks that can be auto-applied to your project with one click.

Plan Mode

Toggle to Plan mode for complex, multi-step tasks. Steve analyzes your request, generates a structured plan, presents it for your approval, then executes code-only output after you approve.

No repetition loops. Steve's proprietary orchestration ensures he moves forward with code generation instead of repeating himself.

Inline Editing — ⌘K

Press ⌘K in the code editor to invoke Steve for inline modifications. Type an instruction like "add error handling" or "convert to async/await" — Steve receives the full file content plus your instruction and returns the modified file in-place.

Vision AI

Drag and drop a UI mockup screenshot into the Steve chat panel. A local Vision AI model analyzes the layout, colors, typography, and components — then generates matching SwiftUI code. The Vision model is loaded on-demand to conserve memory.

Web Research

When Steve encounters a query requiring up-to-date information, he automatically searches the web, fetches and reads relevant pages, and incorporates the results into his response. Fully sandboxed and Mac App Store compatible.

RAG System

Retrieval-Augmented Generation is what makes Steve exceptionally accurate. Kesef Kode ships with a proprietary knowledge base containing over 1.6 million Apple Developer documentation pages.

How It Works

  1. When you ask Steve a question, the RAG engine searches the knowledge base for relevant documentation in real-time
  2. Intelligent query understanding ensures the right SDK documentation is retrieved — even for shorthand or ambiguous terms
  3. The most relevant documentation is provided to Steve as context
  4. Steve's response is grounded in real SDK documentation, not hallucinated API surfaces

Supported Frameworks

SwiftUI, UIKit, AppKit, Combine, Core Data, SwiftData, RealityKit, WidgetKit, async/await concurrency, networking (URLSession), storage (FileManager, Keychain), and more.

Zero extra RAM. The knowledge base is optimized for minimal memory footprint. Searches are near-instant.

Memory System

Steve remembers facts about you across conversations. When he learns something important — your coding style, project architecture, preferences — the memory is automatically extracted, categorized, and synced to your Kesef Plus account.

Advanced Features

MCP Servers

Kesef Kode supports the Model Context Protocol (MCP) — an open standard for connecting AI systems with external tools. Add servers in Settings → MCP and Steve can invoke their tools directly during conversation.

Custom Skills

Teach Steve domain-specific knowledge by creating custom skill files. Skills let you inject your own guidelines, code patterns, and project conventions into Steve's context — globally or per-workspace.

Simulator Manager

Queries xcrun simctl for all available Apple simulators. View, boot, install, and launch apps on any iOS, watchOS, tvOS, or visionOS simulator device.

LLDB Debugger

An integrated LLDB session for low-level debugging. Send commands directly (breakpoint set, po, thread backtrace) with live-streamed output.

Conversation Management

Multiple conversation threads with save, load, and delete. Full message history is preserved across sessions.

Settings

Access via Kesef Kode → Settings (⌘,). Six tabs:

  • General — auto-save interval, default AI mode (Fast/Plan), startup behavior
  • Editor — font size (10–24pt), tab size, line numbers, word wrap, minimap
  • Steve AI — model info, auto-suggest toggle, max tokens (1024/2048/4096), temperature slider (0.1–1.0)
  • MCP — add/remove MCP servers, view connected status and tools
  • Skills — view, create, and delete custom skills
  • Advanced — thermal management, system RAM, LLDB/LSP paths, reset all settings

Architecture

Kesef Kode is built as a native macOS application with a proprietary service-oriented architecture designed for maximum performance and security.

Key Design Principles

  • 100% on-device — all AI inference runs locally on Apple Silicon. Zero data leaves your Mac.
  • Intelligent resource management — AI models are loaded and unloaded dynamically to keep memory usage efficient, even on 16 GB machines.
  • Smart context assembly — Steve's proprietary prompt engine intelligently prioritizes your active file, project structure, and relevant documentation to maximize response quality.
  • Defense-in-depth security — multiple safety layers protect against dangerous shell commands, path traversal, and resource leaks.

Privacy & Security

On-Device Processing

All AI inference runs on your Mac's Apple Silicon. Your code, prompts, and project files never leave your device. No telemetry, no cloud processing, no third-party AI API calls.

Shell Execution Safety

Every AI-initiated command passes through a multi-layer safety system that blocks privilege escalation, destructive filesystem operations, remote code execution, and other dangerous patterns.

Sandbox Protection

When Steve writes code to your project, all file operations are sandboxed within your project directory. Attempts to write outside the project boundary are blocked.

FAQ

Does Kesef Kode require internet?

No. All AI inference runs locally. Internet is only used for: initial model download, web research (when Steve needs up-to-date info), optional memory sync via Kesef Plus account, and Git push/pull.

What models does Steve use?

Steve uses proprietary, Apple Silicon-optimized AI models fine-tuned specifically for Apple platform development. A separate Vision AI model handles UI mockup analysis. Both are included with the app — no downloading or configuration needed.

What Apple platforms can I build for?

All of them: iOS, iPadOS, macOS, watchOS, tvOS, and visionOS. Kesef Kode uses xcodebuild under the hood and supports workspace, project, and Swift Package Manager project types.

Can I use my own models?

Custom model support is planned for a future release.

What is the context window size?

Steve's proprietary prompt engine dynamically manages context to fit within the model's capacity, intelligently prioritizing your active code and most relevant documentation.

How do I report a bug?

Use Help → Report a Bug in the menu bar, or Help → Send Feedback to open a GitHub issue with a pre-filled template.
