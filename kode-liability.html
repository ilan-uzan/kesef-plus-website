Kesef Kode Liability & Licensing

Effective Date: May 10, 2026 · Applies to: Kesef Kode for macOS

This page details your responsibilities when using AI-generated code in Kesef Kode, our third-party license acknowledgments, and data management instructions.

1. Generative Code Liability

Kesef Kode's AI engine ("Steve") is an advisory tool. All code generated, suggested, or modified by Steve is provided for informational and assistive purposes only.

Approving a diff constitutes your acceptance and full responsibility for those changes.

Your Responsibilities

  1. Review Every Diff All AI-generated code passes through a diff review panel. You have full visibility and veto power over every change before it touches disk.
  2. Test Before Deploying Run your test suite after every agent session. AI-generated code may contain logic errors, performance issues, or edge cases.
  3. Audit for Security AI-generated code may inadvertently introduce vulnerabilities. Always perform security review before shipping.
  4. Use Version Control Commit to Git before running the Build Agent. Use the built-in rollback feature for immediate reversal.

What We Disclaim

Data Loss & Corruption

Steve writes files at your approval. Uncommitted changes may be overwritten. Use Git.

Security Vulnerabilities

AI-generated code may contain exploitable patterns. Audit all changes before deployment.

Build & Runtime Failures

Generated code may fail to compile or crash at runtime. The build agent attempts auto-fixes but success is not guaranteed.

App Store Rejection

AI-generated UI or architecture may not meet Apple's review guidelines. Human review is required.

Recommended: Always commit to Git → run agent → review diffs → run tests → then deploy.
